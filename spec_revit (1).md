# Revitチェックアドイン仕様書:電気設備-配管クリアランスチェック

- 作成日:2026-07-15
- 最終更新日:2026-07-15
- ステータス:検討中
- 対象プラットフォーム:Revitアドイン

## 1. 概要・目的

- **このアドインで解決したい課題**
  電気設備(Electrical Fixtures)カテゴリーのファミリ直上に配管が近接して配置されているケースを、モデル上で自動検出する。感電・漏水時の事故リスクや、施工段階での手戻り(是正工事)を設計段階で未然に防ぐ。
- **対象ユーザー・利用シーン**
  設計者(自分自身)による、モデル検証・セルフチェック用ツールとして利用
- **対応Revitバージョン**
  Revit 2027

## 2. 機能要件

### 2.1 主要機能

- 機能A:電気設備カテゴリーと配管カテゴリーの近接干渉チェック
  - 電気設備ファミリの直上に、配管がXY平面上で重なり、かつZ方向に指定距離以内で存在する場合をNGとして検出する

### 2.2 詳細仕様

- **入力**
  - チェック対象モデル(現在開いているRevitプロジェクト、現在のビュー/レベル)
  - 判定距離○m(ユーザーがマニュアル入力、実行時に指定)
- **処理**
  - 電気設備(Electrical Fixtures)カテゴリーの全要素を、現在のビュー/レベルスコープで取得
  - 配管(Pipeカテゴリー全部)の全要素を、現在のビュー/レベルスコープで取得
  - 各電気設備について、XY平面上で重なる配管を抽出(BoundingBoxによる簡易判定)
  - 抽出した配管との垂直距離(Z方向)を計算し、指定距離○m以内かどうかを判定
- **出力**
  - NGとなった電気設備・配管のペア一覧(要素ID、レベル、距離など)
  - 一覧の行をクリックすると該当要素にズーム/選択
- **例外・エラー時の挙動**
  - 未定(「9. 未決定事項・保留事項」参照)

### 2.3 対象外(Non-Goals)

- 電気設備・配管以外のカテゴリー同士の干渉チェック
- 水平方向の離隔チェック(今回は直上・垂直方向のみ)

## 3. UI/UX仕様

- **画面構成・配置**
  - 既存の「アドイン」タブに、本チェックアドイン用のボタンを追加(専用タブは作らない)
  - ボタン押下でシンプルなダイアログを表示(距離○mの入力欄 + 実行ボタンのみ)
- **操作フロー**
  1. Revitで「アドイン」タブ→本アドインのボタンをクリック
  2. ダイアログが開き、判定距離○mを入力
  3. 「実行」ボタンを押すと、現在のビュー/レベルをチェック
  4. チェック完了後、NG結果を一覧表示(要素ID・距離などを含むリスト)
  5. 一覧の行をクリックすると、該当する電気設備・配管の要素がRevit画面上でズーム/選択される
- **参考画面イメージ**
  - なし(今後必要になれば追記)

## 4. 技術構成

- **開発言語・フレームワーク**
  - C# / .NET 8(Revit 2027の標準ターゲットに準拠)
- **アーキテクチャ**
  - IExternalCommandのみで実装(常時起動が必要な処理がないため、IExternalApplicationは不要と判断)
- **Add-inマニフェスト方針**
  - `.addin`ファイルに`AddInType=Command`として1コマンドを登録
  - `<Assembly>` `<ClientId>`(GUID)`<FullClassName>` `<Text>`(ボタン表示名)を定義
- **使用Revit API(想定)**
  - `Autodesk.Revit.DB`:要素取得(`FilteredElementCollector`)、BoundingBox取得
  - `Autodesk.Revit.UI`:コマンド登録、ダイアログ表示、要素の選択・ズーム(`UIDocument.ShowElements`など)
  - カテゴリフィルタ:`BuiltInCategory.OST_ElectricalFixtures`、`BuiltInCategory.OST_PipeCurves`
- **ローカル開発環境**
  - 現状未整備。今後、Revit 2027 SDK・Visual Studio・NuGetパッケージ(`Revit_All_Main_Versions_API_x64`等)の準備が必要

## 5. データ構造・扱う対象

- **チェック対象要素**
  - 電気設備(`BuiltInCategory.OST_ElectricalFixtures`)
  - 配管(`BuiltInCategory.OST_PipeCurves`)
- **チェック範囲**
  - 現在開いているビュー/レベルのみが対象(プロジェクト全体は対象外)
  - `FilteredElementCollector`を現在のビュー(`Document.ActiveView`)にスコープして要素取得
- **XY重なり判定のロジック**
  - 各要素の`BoundingBox`(XYZ min/max)を取得
  - 電気設備・配管それぞれのBoundingBoxをXY平面(X, Y座標)に投影し、矩形同士が重なっているかを判定(簡易的な干渉チェック)
  - 重なりが確認できた場合のみ、Z方向の距離計算に進む
- **Z方向距離判定**
  - 電気設備BoundingBoxの上端Z座標と、配管BoundingBoxの下端Z座標の差を計算
  - その差が入力された判定距離○m以内であればNGと判定
- **出力データ項目(NGリスト用)**
  - 電気設備の要素ID・ファミリ名・レベル
  - 配管の要素ID・系統名・レベル
  - 実際の垂直距離(計算値)

## 6. 外部連携

- 対象外(Revit内で完結する想定。外部システム・Excel出力等の連携は行わない)

## 7. 制約事項・非対応範囲

- 対応Revitバージョン:Revit 2027のみ(2024〜2026等の旧バージョンは非対応)
- チェック範囲は現在開いているビュー/レベルのみ(プロジェクト全体の一括チェックは非対応)
- XY重なり判定はBoundingBox(簡易判定)による近似計算であり、実際の正確な形状(配管の断面形状や電気設備の詳細形状)による厳密な干渉判定ではない
- 電気設備・配管以外のカテゴリー同士の干渉は対象外
- 水平方向の離隔チェックは対象外(垂直方向・直上のみ)
- Excel等への外部エクスポート機能はなし(Revit内の一覧表示・要素選択のみ)

## 8. 検討時の決定事項と背景

| 決定事項 | 背景・理由 |
|---|---|
| 判定距離○mはマニュアル入力とする | 現場やプロジェクトによって求められる離隔距離が異なるため、固定値ではなく実行時に指定できるようにした |
| XY重なり判定はBoundingBox(簡易判定)を採用 | 正確な形状判定より実装がシンプルで、処理速度も速いため。今回のスコープでは十分と判断 |
| チェック範囲は現在開いているビュー/レベルのみ | プロジェクト全体の一括チェックは処理負荷や実装難易度が上がるため、まずは限定範囲でシンプルに実装する方針とした |
| UIは既存の「アドイン」タブにボタン追加+シンプルダイアログ | 専用タブやドッキングパネルは今回不要と判断。操作をシンプルに保つため |
| アーキテクチャはIExternalCommandのみ | 常時起動が必要な処理がなく、ボタン実行型で十分なため |
| 配管側はPipeカテゴリー全部を対象 | 現時点で系統を絞る必要性がないため、まず全部を対象とした |

## 9. 未決定事項・保留事項

- エラー・例外時の挙動(電気設備や配管が1件も存在しない場合の表示など)
- 判定距離○mの入力形式(整数のみ/小数対応、単位はmで固定か)、および入力値のバリデーション(0以下やマイナス値の扱い)
- 現在のビューが3Dビュー/平面図以外(断面図やスケジュールなど、要素取得に適さないビュー)の場合の挙動
- 大規模プロジェクトでの処理速度・パフォーマンス対応の要否
- 将来的な機能拡張(プロジェクト全体チェック、Excel出力など)の要否

  - `Element.get_BoundingBox(View)`

- **開発環境セットアップで必要になりそうなもの**
  - Visual Studio 2022以降
  - Revit 2027 SDK(またはNuGetパッケージ経由のAPI参照)
  - `.addin`マニフェストの配置先:`%APPDATA%\Autodesk\Revit\Addins\2027\`
